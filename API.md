# Even G2 Gateway API Specification

Status: current implementation

This document describes the HTTP and WebSocket interfaces currently implemented by the gateway server in this repository.

## 1. Overview

- Default base URL: http://127.0.0.1:8765
- Default WebSocket endpoint: /ws
- Content type for HTTP request bodies: application/json
- Authentication: none
- Response format: JSON for API endpoints

The gateway exposes three kinds of interfaces:

1. HTTP endpoints for display control, microphone control, synthetic touch injection, and status inspection.
2. A WebSocket endpoint for real-time status and device events.
3. Static UI assets served from the same process.

## 2. Conventions and Runtime Notes

### 2.1 Event Envelope

All WebSocket events use the same envelope:

```json
{
  "seq": 42,
  "kind": "glasses.touch",
  "timestamp": "2026-05-26T12:34:56.789Z",
  "data": {
    "gesture": "single_tap",
    "source": 0
  }
}
```

Fields:

- seq: monotonically increasing event sequence number generated by the server process.
- kind: event type string.
- timestamp: UTC ISO-8601 timestamp with millisecond precision.
- data: event-specific payload.

### 2.2 Error Responses

When an HTTP request is rejected, the server returns JSON like this:

```json
{
  "accepted": false,
  "error": "human-readable message"
}
```

Typical status codes:

- 400 for validation or payload errors.
- 500 for runtime failures.

### 2.3 Display and Text Rules

The current implementation enforces these input rules:

- Any incoming text field must be 1000 bytes or fewer after UTF-8 encoding.
- Empty text is not sent to the glasses as an empty string. It is normalized to a single space character.
- If clear is true, it takes precedence over text or elements.
- For image or mixed layout requests, gamma and dither may be supplied at the top level.
- If a layout contains no text elements, the gateway injects a full-screen text capture container internally.
- At most one text element may set capture_events to true.
- If no text element sets capture_events, the first text element becomes the capture target automatically.
- Large logical images may be split into internal tiles, but the final image container count must not exceed 4.

### 2.4 WebSocket Delivery Behavior

- Every newly connected WebSocket client immediately receives one status.snapshot event.
- Events are broadcast to all connected WebSocket clients.
- Each WebSocket client has a bounded outgoing queue of 128 events.
- Slow clients are disconnected if their queue overflows.
- glasses.raw_packet events are only emitted when the server is started with debug raw packet output enabled.

## 3. HTTP API

### 3.1 GET /api/status

Returns the current server status and the latest known glasses state.

Example response:

```json
{
  "server": {
    "host": "0.0.0.0",
    "port": 8765,
    "websocket_path": "/ws",
    "static_dir": "ui"
  },
  "glasses": {
    "phase": "ready",
    "ready": true,
    "search_id": "",
    "last_serial_number": "G2_123456789012",
    "mic_enabled": false,
    "target_mic_enabled": false,
    "battery_level": 84,
    "charging": false,
    "left_firmware_version": "...",
    "right_firmware_version": "...",
    "firmware_version": "...",
    "last_error": "",
    "last_gesture": "single_tap",
    "left": {
      "side": "left",
      "address": "AA:BB:CC:DD:EE:01",
      "mac_address": "AA:BB:CC:DD:EE:01",
      "name": "G2_..._L_...",
      "connected": true,
      "authenticated": true
    },
    "right": {
      "side": "right",
      "address": "AA:BB:CC:DD:EE:02",
      "mac_address": "AA:BB:CC:DD:EE:02",
      "name": "G2_..._R_...",
      "connected": true,
      "authenticated": true
    },
    "page": {
      "startup_page_created": true,
      "page_created": true,
      "page_has_text_container": true,
      "page_has_fullscreen_text_container": true,
      "text_container_id": 1,
      "current_text_content": "Hello",
      "has_cached_display_request": true
    }
  }
}
```

Notes:

- phase is one of: idle, scanning, connecting, initializing, ready, recovering, stopped.
- last_error contains the latest connection/runtime error, if any.
- last_gesture is updated whenever a touch gesture is observed or injected.

### 3.2 POST /api/display

Sends text, image, or mixed layout content to the glasses.

Success response:

```json
{
  "accepted": true,
  "mode": "fast_text",
  "queued": true
}
```

The mode field is one of:

- fast_text
- layout
- clear

#### 3.2.1 Fast Text Request

Use this form for plain full-screen text without positioning.

```json
{
  "text": "Hello from the gateway"
}
```

Behavior:

- The gateway uses the fast text path.
- The text is sent through the internal text update queue.
- If the current page is not already a full-screen text container page, the gateway rebuilds it.
- If text is an empty string, it is normalized to a single space.

Current implementation limit:

- text must be 1000 UTF-8 bytes or fewer.

#### 3.2.2 Clear Request

```json
{
  "clear": true
}
```

Behavior:

- Clears the cached display request.
- Replaces the visible content with an empty-looking page backed by a single-space text payload.
- Does not intentionally drop page ownership.

#### 3.2.3 Layout Request

Use this form for positioned text, images, or mixed content.

```json
{
  "gamma": 1.0,
  "dither": false,
  "elements": [
    {
      "type": "text",
      "text": "Header",
      "x": 0,
      "y": 0,
      "width": 320,
      "height": 60,
      "border_width": 0,
      "border_color": 0,
      "border_radius": 0,
      "padding": 4,
      "capture_events": false,
      "container_name": "header"
    },
    {
      "type": "image",
      "image_base64": "...",
      "x": 0,
      "y": 80,
      "width": 200,
      "height": 100
    }
  ]
}
```

Top-level optional fields:

- gamma: image gamma correction factor. 1.0 means no correction.
- dither: if true, 4-bit Floyd-Steinberg dithering is applied to image rendering.

Text element fields:

- type: must be text.
- text: string, required.
- x, y, width, height: integer layout bounds.
- border_width, border_color, border_radius: optional integers.
- padding: optional integer.
- capture_events: optional boolean.
- container_name: optional string.

Image element fields:

- type: must be image.
- image_base64: required string, either raw Base64 or a data URL.
- x, y, width, height: required integers.

Validation and normalization rules:

- Unsupported type values are rejected.
- At most one text element may set capture_events to true.
- If no text elements exist, the gateway injects an internal full-screen capture text container with a single-space payload.
- If text elements exist and none has capture_events=true, the first text element becomes the capture target.
- Each text field must be 1000 UTF-8 bytes or fewer.
- Image tiles must fit within internal G2 image container constraints after splitting.
- If the generated internal image tile count exceeds 4, the request is rejected.

#### 3.2.4 Single-Image Shorthand

Instead of elements, the server also accepts a shorthand single-image request:

```json
{
  "image_base64": "...",
  "x": 0,
  "y": 0,
  "width": 200,
  "height": 100,
  "gamma": 0.9,
  "dither": true
}
```

This is internally converted into a one-element layout request.

### 3.3 POST /api/mic

Enables or disables the glasses microphone.

Request:

```json
{
  "enabled": true
}
```

Success response:

```json
{
  "accepted": true,
  "enabled": true
}
```

Validation:

- enabled must be a JSON boolean.

Behavior:

- The target mic state is updated immediately.
- If the glasses are already ready, the gateway ensures a startup page exists and then sends audio control.
- If the glasses are not ready yet, the desired state is remembered and restored after initialization or recovery.

### 3.4 POST /api/touch

Injects a synthetic touch event into the server event stream.

This endpoint is intended for testing, browser-side simulation, and automation.

Request:

```json
{
  "gesture": "single_tap"
}
```

Allowed gesture values:

- single_tap
- double_tap
- swipe_up
- swipe_down

Success response:

```json
{
  "accepted": true,
  "gesture": "single_tap"
}
```

Behavior:

- A glasses.touch event is broadcast over WebSocket with source set to http.
- last_gesture in the status snapshot is updated.

## 4. WebSocket API

### 4.1 Endpoint

- GET /ws by default
- The actual path is configurable through server.websocket_path in the YAML configuration.

### 4.2 Initial Event

Immediately after a WebSocket connection is accepted, the server sends one status.snapshot event.

### 4.3 Event Types

#### status.snapshot

Payload data is exactly the same shape as GET /api/status.

```json
{
  "kind": "status.snapshot",
  "data": {
    "server": { "host": "0.0.0.0", "port": 8765, "websocket_path": "/ws", "static_dir": "ui" },
    "glasses": { "phase": "ready", "ready": true }
  }
}
```

#### connection.state

Emitted when the connection phase changes.

Example:

```json
{
  "kind": "connection.state",
  "data": {
    "phase": "recovering",
    "ready": false,
    "left_connected": false,
    "right_connected": false,
    "reason": "left_disconnected"
  }
}
```

The reason field is only present on some recovery-related transitions.

#### glasses.touch

Emitted for real touch gestures from the glasses and for synthetic gestures injected through POST /api/touch.

Examples:

```json
{
  "kind": "glasses.touch",
  "data": {
    "gesture": "single_tap",
    "source": 0
  }
}
```

```json
{
  "kind": "glasses.touch",
  "data": {
    "gesture": "single_tap",
    "source": "http"
  }
}
```

Observed gesture names include:

- single_tap
- double_tap
- swipe_up
- swipe_down
- foreground_enter
- foreground_exit
- system_exit

#### glasses.authentication

Emitted when a lens authentication response is received.

```json
{
  "kind": "glasses.authentication",
  "data": {
    "source": "L",
    "authenticated": true
  }
}
```

#### glasses.battery

Emitted when battery information is parsed from a G2 setting response.

```json
{
  "kind": "glasses.battery",
  "data": {
    "level": 84,
    "charging": false
  }
}
```

#### glasses.firmware

Emitted when firmware version information is parsed.

```json
{
  "kind": "glasses.firmware",
  "data": {
    "left": "1.2.3",
    "right": "1.2.3"
  }
}
```

#### glasses.dashboard

Emitted when a menu/dashboard app selection event is received from the glasses.

```json
{
  "kind": "glasses.dashboard",
  "data": {
    "action": "selected",
    "app_id": 10029
  }
}
```

#### glasses.mic_audio

Emitted for incoming microphone audio frames.

```json
{
  "kind": "glasses.mic_audio",
  "data": {
    "source": "R",
    "frame_size": 200,
    "data_base64": "..."
  }
}
```

Notes:

- Duplicate consecutive audio frames are suppressed.
- frame_size is the actual usable bytes forwarded in the current event.

#### glasses.raw_packet

Only emitted when raw packet debugging is enabled at server startup.

```json
{
  "kind": "glasses.raw_packet",
  "data": {
    "source": "R",
    "size": 48,
    "hex": "aa2101..."
  }
}
```

#### system.error

Emitted when the connection loop or runtime logic encounters an error.

```json
{
  "kind": "system.error",
  "data": {
    "message": "could not discover left and right G2 devices during scan"
  }
}
```

#### system.reinitialize

Emitted when the gateway rebuilds its runtime page state after a page shutdown or system exit.
The existing BLE connection is kept; this event does not imply a BLE reconnect.

```json
{
  "kind": "system.reinitialize",
  "data": {
    "reason": "page_shutdown"
  }
}
```

## 5. Static UI Endpoints

These are not JSON API endpoints, but they are served by the same process:

- GET /
- GET /app.js
- GET /styles.css

## 6. Implementation Notes and Current Limitations

- The server does not implement authentication or authorization.
- The current implementation rejects incoming text larger than 1000 UTF-8 bytes instead of using a placeholder-plus-upgrade fallback.
- The WebSocket API is broadcast-only. Clients do not send commands over WebSocket.
- Image rendering options gamma and dither are applied only to image/layout requests.
- Runtime behavior depends on actual BLE availability and the presence of both left and right G2 lenses.
